# Data sources & files

Every piece of information this project uses, where it comes from, and where
it lands on disk. All external sources are free, community/hobbyist services —
this project self-throttles (see [Rate limiting](#rate-limiting)) and pulls
**metadata only**, never the copyrighted SID audio itself.

---

## External sources

### 1. DeepSID API — the primary source

- **Base URL:** <https://deepsid.chordian.net/api/v1.php>
- **Source reference:** <https://github.com/Chordian/deepsid/blob/master/api/v1.php>
- **Local reference doc:** [`docs/DEEPSID-API.md`](DEEPSID-API.md)
- **Client:** `scripts/lib/deepsid-client.js` (JSON) · **used by:** `fetch-players.js`, `fetch-composers.js`

| Endpoint | Returns | Notes |
|---|---|---|
| `?players` | Full players/editors database (129 curated) | → `data/players.json` |
| `?profile=<path>/` | Composer profile (bio, photo, country, dates) | → `data/composers/*.json` |
| `?file=<path>.sid` | Single SID file metadata | ⚠ often down — see below |
| `?file=<path>/` | Folder metadata + file array | ⚠ often down |
| `?folder=<path>/` | Folder metadata + subfolders only | ⚠ often down |

> The `?file=`/`?folder=` endpoints have been unreliable (`"Could not connect
> to the DeepSID database"`). Per-file listings are largely served instead
> from the **DeepSID database export** (below), falling back further to HVSC's
> STIL.txt. `?players` and `?profile=` still work live.

Composer thumbnails/brand images referenced from profiles:
<https://deepsid.chordian.net/images/composers/>,
`…/images/composer.png`, `…/images/brands/`.

### 2. DeepSID database export — offline bulk import

- **Where:** the "Setting up for offline use" section of
  <https://github.com/Chordian/deepsid> (`DeepSID_Database.zip`)
- **Unpacked to:** `deepsid_dl/DeepSID_Database/` (gitignored, manual)
- **Importer:** `scripts/import-deepsid-dump.js` (mysqldump parser in
  `scripts/lib/sql-dump.js`) · **one-time, dated snapshot**
- **Produces:** `data/composer-list.json` (the ~1,895-composer seed) and
  per-file `player` data; snapshot version recorded in
  `data/deepsid-dump/meta.json`.

### 3. SIDId — player/editor identification database

- **File:** `sidid.nfo` — ships **inside the DeepSID offline bundle** above
  (`deepsid_dl/sidid.nfo`, gitignored). This is the SIDId project's
  human-readable index that maps a raw player tag → NAME / AUTHOR / RELEASED /
  REFERENCE / COMMENT (playback technique).
- **Encoding:** ISO-8859-1 (not UTF-8).
- **Importer:** `scripts/import-sidid.js` (parser: `scripts/lib/sidid.js`)
- **Produces:** `data/sidid.json`. Fills in author/name/year/reference/
  technique for the **inferred** players DeepSID's own DB never wrote up.

### 4. CSDb webservice — scener & release enrichment

- **Base URL:** <https://csdb.dk/webservice/> (documented at that URL)
- **Client:** `scripts/lib/csdb-client.js` (XML, via `fast-xml-parser`)

| Endpoint | Used for | Script | Output |
|---|---|---|---|
| `?type=scener&id=<id>` | Composer bios, group history, scene roles | `fetch-csdb.js` | `data/csdb/<slug>.json` |
| `?type=release&id=<id>` | Player screenshot, **credits**, release date, releasing group, download link | `fetch-player-media.js` | `data/csdb/players.json` |
| `?type=group&id=<id>` | (available; group detail) | — | — |

`fetch-player-media.js` fetches both each curated player's `csdb_id` **and**
every SIDId `reference` that points at a CSDb release (so inferred players get
release data too). Release/download links seen in data:
`csdb.dk/release/?id=…`, `csdb.dk/release/download.php?id=…`.

### 5. HVSC — community-verified documentation (metadata only)

- **Base:** <https://www.hvsc.c64.org/>
- **Fetcher/parser:** `scripts/fetch-hvsc-docs.js` (plain text, decoded as
  **ISO-8859-1**) · no client needed

| File | URL | Purpose | Output |
|---|---|---|---|
| `Musicians.txt` | <https://www.hvsc.c64.org/download/C64Music/DOCUMENTS/Musicians.txt> | Verified handle / real name / group / country | `data/hvsc/musicians.json` |
| `STIL.txt` | <https://www.hvsc.c64.org/download/C64Music/DOCUMENTS/STIL.txt> | Per-file title/artist + free-text song comments; fallback file listing | `data/hvsc/stil.json` (+ raw `STIL.txt`) |

> HVSC does **not** ship a player/editor database — the player identification
> in the HVSC/DeepSID world is produced by the SIDId tool (source #3). HVSC's
> contribution here is composer verification and per-song notes only.

### Upstream gap reporting (output, not a source)

- Documentation gaps are meant to be filed at
  <https://github.com/Chordian/deepsid/issues> (see `find-gaps.js`,
  [`docs/GAPS_REPORT.md`](GAPS_REPORT.md)).

## Local data files

Everything lands in `data/`. Only **`data/composer-list.json`** is committed
(the seed); all other data files are **gitignored** and regenerated by the
scripts above (`npm run all`, or the individual `fetch:*` / `import:*` scripts).

| Path | Source | Committed? |
|---|---|---|
| `data/composer-list.json` | DeepSID database export (seed) | ✅ **yes** |
| `data/composers/*.json` | DeepSID `?profile=` (per composer) | gitignored |
| `data/players.json` | DeepSID `?players` | gitignored |
| `data/sidid.json` | SIDId `sidid.nfo` | gitignored |
| `data/csdb/<slug>.json` | CSDb `?type=scener` (per composer) | gitignored |
| `data/csdb/players.json` | CSDb `?type=release` (per player/release) | gitignored |
| `data/hvsc/musicians.json` | HVSC `Musicians.txt` | gitignored |
| `data/hvsc/stil.json`, `data/hvsc/STIL.txt` | HVSC `STIL.txt` | gitignored |
| `data/deepsid-dump/meta.json` | DeepSID export snapshot version marker | gitignored |
| `data/gaps-report.json` | `find-gaps.js` (derived) | gitignored |
| `deepsid_dl/` (bundle: `DeepSID_Database/`, `sidid.nfo`, images) | manual download from github.com/Chordian/deepsid | gitignored |

`build-html.js` reads only from `data/` and injects it into
`templates/index.html.template` → `output/index.html` (also gitignored).

---

## Rate limiting

Both DeepSID and CSDb clients self-throttle to **1 request per 400 ms**,
deliberately conservative — these are free services, not commercial APIs. See
`scripts/lib/deepsid-client.js` and `scripts/lib/csdb-client.js`.
